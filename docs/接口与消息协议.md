# 接口与消息协议

## 1. REST API（后端）
项目根目录提供 OpenAPI 文件：
- `api-docs.yaml`

建议在后端启动后使用 Swagger UI 浏览（通常可用其一）：
- `http://localhost:8080/swagger-ui/index.html`
- `http://localhost:8080/swagger-ui.html`

### 1.1 设备管理
- `GET  /api/v1/devices`：获取所有无人机
- `POST /api/v1/devices`：新增无人机
- `GET  /api/v1/devices/{id}`：获取单台无人机
- `PUT  /api/v1/devices/{id}`：更新无人机
- `DELETE /api/v1/devices/{id}`：删除无人机

### 1.2 任务管理
- `GET  /api/v1/tasks`：任务查询（支持条件）
- `POST /api/v1/tasks`：创建任务（含航线点 routePoints）
- `GET  /api/v1/tasks/{id}`：任务详情
- `PUT  /api/v1/tasks/{id}`：更新任务
- `DELETE /api/v1/tasks/{id}`：删除任务
- `PUT  /api/v1/tasks/{id}/start`：启动任务
- `PUT  /api/v1/tasks/{droneId}/finish`：结束任务（注意参数为 droneId）

### 1.3 数据上报
- `POST /api/v1/drones/upload`：上传路面图片（multipart/form-data）
  - query/form 参数：taskId、droneId、lng、lat、altitude、yaw、pitch、roll、fov
  - file：图片二进制
- `POST /api/v1/drones/telemetry`：上报遥测数据（JSON Map）

### 1.4 地图与详情
- `GET /api/v1/map/tile?z=..&x=..&y=..`：瓦片范围内实体点位
- `GET /api/v1/map/{entityId}/details`：实体历史详情（含图片URL）

## 2. WebSocket 协议
连接地址（默认）：
- `ws://localhost:8080/ws/telemetry`

消息示例（后端广播给前端）：
1) 遥测消息
- type: telemetry
- data: （当前实现为数组/对象，具体字段随无人机上报而定）

2) 新病害告警
- type: new_defect
- data:
  - taskId, lng, lat
  - detail: AI 返回结构（detections 等）
  - imageUrl: 结果图可访问 URL

3) 任务启动广播
- type: task_start
- data: task 对象（包含 routePoints）
