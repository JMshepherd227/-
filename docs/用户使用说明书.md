# 用户使用说明书

## 1. 访问方式
- 打开前端：`http://localhost:5173`
- 系统顶部导航包含：地图、设备、任务、采集

## 2. 功能说明

### 2.1 地图模块
目标：在地图上查看病害点位、点击查看病害详情，观察无人机轨迹与实时状态。
操作：
1. 打开“地图”页，等待高德地图加载完成
2. 点击“刷新”手动加载视口内瓦片数据；或开启“自动刷新”
3. 地图中出现点位后，可点击点位查看：
   - 实体ID/类型/坐标/时间
   - 识别详情列表（类型、置信度、地址、结果图/原图链接）

说明：
- 地图点位数据来自后端 `/api/v1/map/tile`（瓦片加载）
- 详情数据来自后端 `/api/v1/map/{entityId}/details`

### 2.2 设备管理
目标：维护无人机设备信息，供任务绑定与演示。
操作：
1. 进入“设备”
2. 点击“新增无人机”，填写名称并保存
3. 可对设备进行编辑与删除

说明：
- 状态字段用于表示无人机当前是否空闲/任务中（以系统内部枚举为准）

### 2.3 任务管理（含航线点位）
目标：创建巡检任务、配置航线点、启动/结束任务。
操作：
1. 进入“任务”，点击“新建任务”
2. 填写任务名称、选择绑定无人机
3. 点击“开始选点”，在地图上逐点点击生成航线点位（支持地址反查显示）
4. 保存任务后，列表中可看到任务状态与病害数量
5. 点击“启动”下发任务（系统会通过 WebSocket 广播 task_start）
6. 执行完成后点击“结束”（注意结束接口以 droneId 为参数）

说明：
- 任务创建/修改：`/api/v1/tasks`（POST/PUT）
- 任务启动：`/api/v1/tasks/{id}/start`
- 任务结束：`/api/v1/tasks/{droneId}/finish`（这里的路径参数是无人机ID）

### 2.4 采集控制台（实时事件）
目标：观察 WebSocket 实时事件，包括遥测广播、病害告警等。
操作：
1. 进入“采集”
2. 点击“连接 WebSocket”
3. 观察日志列表：
   - `telemetry`：无人机遥测
   - `new_defect`：发现新病害告警（可点击结果图链接）
4. 可“清空日志”重新观察

说明：
- WebSocket 默认路径：`ws://localhost:8080/ws/telemetry`
- 病害告警由后端识别入库后触发事件推送

## 3. 典型演示流程（建议）
1. 新增 1 台无人机
2. 创建 1 条巡检任务（点选 3-6 个航线点）
3. 启动任务
4. 通过接口模拟上传 3-10 张巡检图片（含经纬度/高度/姿态参数）
5. 地图刷新查看病害点位与详情
6. 采集控制台连接 WebSocket，展示实时遥测与告警

## 4. 注意事项
- 若 AI 未识别到病害，地图不会产生点位（属于正常情况）
- 若图片/模型/路径配置不一致，可能导致“结果图无法打开”
- 演示前建议清空/重置数据库，确保数据口径一致（详见 06）
