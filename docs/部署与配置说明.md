# 部署与配置说明

## 1. 前端配置（frontend/.env.local）
关键变量：
- `VITE_API_BASE`：后端地址（默认 `http://localhost:8080`）
- `VITE_WS_BASE`：WebSocket 地址（默认 `ws://localhost:8080/ws/telemetry`）
- `VITE_AMAP_KEY`：高德地图 Web Key
- `VITE_AMAP_SECURITY_JS_CODE`：高德安全密钥（安全 JS Code）

说明：
- 前端地图依赖高德 JS SDK，Key 配置不正确会导致地图初始化失败。

## 2. 后端配置（backend/src/main/resources/application.yml）
关键项：
- `spring.datasource.*`：MySQL 连接信息
- `spring.data.redis.*`：Redis 连接信息
- `drone.origin-dir`：原图保存目录（本机路径）
- `drone.root-dir`：项目根目录（用于拼接图片相对路径）
- `drone.ai-url`：AI 检测接口（默认 `http://localhost:8000/predict/`）
- `drone.match-url`：SIFT 单应矩阵接口（默认 `http://localhost:8000/get_homography/`）
- `drone.LoFTR-url`：LoFTR 单应矩阵接口（默认 `http://localhost:8000/get_homography_loftr/`）
- `drone.GNN-url`：病害点匹配接口（默认 `http://localhost:8000/match_points/`）
- `amap.key`：用于后端逆地理编码（RoadService）

静态资源映射：
- 后端通过 `WebConfig` 将：
  - `/origin/**` 映射到本地 `origin` 文件夹
  - `/result/**` 映射到本地 `result` 文件夹
用于在前端直接访问图片 URL。

务必检查：
- 你机器上是否存在 `origin` / `result` 目录，并且路径与配置一致。
- 若路径不同，请修改 `application.yml` 或 `WebConfig`。

## 3. AI 服务配置（ai-service/app.py）
当前版本存在若干硬编码路径（需与你机器一致）：
- YOLO 权重路径：`.../ai-service/model/best.pt`
- GNN 权重路径：`.../ai-service/model/best_matcher.pt`
- 结果图输出目录：`.../result/<date>/xxx.jpg`
- 直接把这些路径改成你本机实际路径即可

## 4. 端口与连通性检查
- 前端 -> 后端：HTTP（默认 8080）
- 前端 -> 后端：WebSocket（默认 `/ws/telemetry`）
- 后端 -> AI 服务：HTTP（默认 8000）
- 后端 -> Redis：6379
- 后端 -> MySQL：3306

## 5. 推荐部署顺序
1. MySQL（导入表结构与数据）
2. Redis
3. AI 服务（FastAPI）
4. 后端（Spring Boot）
5. 前端（Vue3）

## 6. 安全注意事项（必须）
- 请勿在公开仓库中提交真实的数据库密码、地图 Key 等敏感信息。
- 当前仓库内已出现明文配置的风险点（如 application.yml），建议提交前做脱敏处理。
